Key Components

Tournament Engine

Manages the full tournament lifecycle: registration, Swiss-system bracket generation (Dutch variant), round advancement, and finalization with placement calculation.

Swiss Pairing Algorithm

Players are sorted by score (wins), then ELO. Pairing proceeds top-down within score groups, avoiding rematches. Odd player counts are handled with byes.

ELO Calculator

Standard ELO formula with K-factor: K=32 for new players (<20 matches), K=16 for established players. History is recorded for every rating change.

Badge Engine

Checks achievement conditions after each match and tournament completion. Awards badges automatically and prevents duplicates.

Scheduler

Coroutine-based scheduler that triggers weekly tournament registration, bracket generation, daily summaries, and weekly recaps based on configured timezone and times.

Database Schema

Core tables: players, tournaments, tournament_participants, tournament_rounds, matches, match_sets, elo_history, badges.

Migrations are managed with numbered SQL files in db/migrations/, tracked in a schema_version table.
